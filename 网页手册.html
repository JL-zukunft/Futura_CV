EdgeOne Pages 部署

1

注册并登录

访问 EdgeOne 控制台，使用 GitHub 账号登录

2

连接 GitHub 仓库

点击"创建项目" → 选择"从 GitHub 导入" → 授权并选择您的 Futura_CV 仓库

3

配置构建参数

构建命令：无需配置（静态网站） 输出目录：/ 部署分支：main
4

等待部署完成

等待 1-2 分钟，部署成功后会显示访问地址

✅ 成功标志： 状态显示"部署成功"，并显示访问域名

后端配置（腾讯云）

1

创建云开发环境

访问 腾讯云开发控制台 → 开通云开发 → 创建环境（选择免费版）

2

修改配置文件

打开 backend-tencent/cloudbaserc.json，将 envId 改为您的环境 ID

{ "envId": "您的环境 ID", "functionRoot": "api", "functions": { "portfolio-api": { "handler": "index.main", "runtime": "Nodejs16.13" } } }
3

安装依赖并部署

# 进入后端目录 cd backend-tencent # 安装依赖 npm install # 登录腾讯云 cloudbase auth:login # 部署云函数 cloudbase functions:deploy portfolio-api
4

配置 API 触发器

在云开发控制台 → 云函数 → portfolio-api → 触发管理 → 创建触发器（API 网关）

📝 记录 API 地址： 格式类似 https://您的环境 ID.service.tcloudbase.com/api

数据库配置

1

创建数据库集合

在云开发控制台 → 数据库 → 创建以下 9 个集合：

cases - 案例主表 caseTabs - 案例详情标签页 caseImages - 案例图片 visitors - 访客记录 pageViews - 页面浏览记录 caseViews - 案例浏览记录 translations - 多语言翻译 siteConfig - 网站配置 aiGenerations - AI 生成记录
2

执行初始化脚本

# 运行初始化脚本 node scripts/init-database.js
3

设置数据库权限

公开内容（cases 等）：设置为"所有用户可读"

私密数据（visitors 等）：设置为"仅创建者可读写"

添加案例内容

💡 方法一：通过控制台添加（推荐）
1

进入数据库控制台

云开发控制台 → 数据库 → 选择集合（cases）

2

添加案例数据

{ "title": "我的案例标题", "subtitle": "案例副标题", "description": "案例描述内容", "cover_image": "/images/cover.jpg", "order_num": 1, "is_published": true }
3

添加案例详情

在 caseTabs 集合中添加：

{ "case_id": 1, "tab_name": "overview", "tab_label": "项目概览", "content": "<h3>项目背景</h3><p>...</p>" }

常见问题

Q1: 部署后看不到案例数据？

A: 检查以下几点：

  • API URL 是否正确修改
  • 数据库是否创建了对应的集合
  • 是否插入了初始数据
  • 打开浏览器 F12 查看 Console 错误

Q2: 云函数部署失败？

A: 常见原因：

  • 没有安装依赖：npm install
  • 配置文件中的环境 ID 错误
  • 没有登录：cloudbase auth:login
  • 查看错误日志，根据提示修复

Q3: 费用会超吗？

A: 个人使用完全不会超！

  • EdgeOne Pages：免费额度充足
  • 腾讯云：5GB 存储 + 500 万读写/月
  • GitHub Pages：无限流量
  • 总计：¥0/月

Q4: 如何修改网站标题和描述？

A: 打开 index.html，找到以下代码并修改：

<title>Futura | 消费品 × AI</title> <meta name="description" content="消费品×AI 产品设计师">
