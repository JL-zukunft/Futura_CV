## 入门问题

### Q1: EasyFolio 需要编程知识吗？

**A:** 不需要！EasyFolio 设计为零代码工具。你只需要：
- 准备好你的个人信息
- 按照指南编辑配置文件
- 部署到免费平台

如果你愿意，也可以深入定制代码，但这完全是可选的。

---

### Q2: 我可以免费使用 EasyFolio 吗？

**A:** 完全免费！
- EasyFolio 本身是开源免费的
- 部署平台（GitHub Pages、Vercel、Netlify）都有免费额度
- 没有隐藏费用

---

### Q3: 多久能完成第一个作品集？

**A:** 按照快速入门指南，10分钟内可以完成并部署上线。

---

### Q4: EasyFolio 适合什么人群？

**A:** 适合：
- 🎨 **设计师** - 展示作品
- 💻 **开发者** - 展示项目和技能
- 🎯 **求职者** - 打造专业形象
- 📈 **自由职业者** - 建立个人品牌

## 配置问题

### Q5: 如何修改个人信息？

**A:** 有两种方式：

**方式一：使用 config.json（推荐）**
1. 创建或编辑 `config.json`
2. 按照 DATA_TEMPLATE.md 的格式填入信息
3. 重新生成或手动更新 index.html

**方式二：直接编辑 index.html**
1. 用文本编辑器打开 `index.html`
2. 找到对应的内容部分
3. 直接修改文字

---

### Q6: 可以添加多个技能吗？

**A:** 当然可以！在 `config.json` 的 `skills` 数组中添加任意数量的技能：

```json
{
  "skills": [
    {
      "name": "React",
      "level": "专家",
      "category": "前端"
    },
    {
      "name": "Node.js",
      "level": "进阶",
      "category": "后端"
    },
    {
      "name": "UI设计",
      "level": "入门",
      "category": "设计"
    }
  ]
}
```

---

### Q7: 技能的"级别"有什么用？

**A:** 技能级别用于展示你的熟练程度：
- **入门** - 刚接触，了解基础
- **进阶** - 有一定经验，能独立完成任务
- **专家** - 精通，能指导他人

你可以根据实际情况选择。

---

### Q8: 可以不显示某些部分吗？

**A:** 可以！直接编辑 `index.html`，注释掉或删除不需要的部分即可。

---

## 部署问题

### Q9: 哪个部署平台最好？

**A:** 推荐顺序：
1. **GitHub Pages** - 最简单，与 GitHub 完美集成（⭐⭐⭐⭐⭐）
2. **Vercel** - 功能强大，体验极佳（⭐⭐⭐⭐）
3. **Netlify** - 配置灵活（⭐⭐⭐⭐）
4. **EdgeOne Pages** - 适合国内用户（⭐⭐⭐）

新手建议从 GitHub Pages 开始。

---

### Q10: 部署后多久能生效？

**A:** 通常 1-3 分钟。如果是第一次部署，可能需要稍长一些时间。

---

### Q11: 部署后显示 404 怎么办？

**A:** 检查以下几点：
1. 确认部署的分支正确（main/master）
2. 确认 index.html 在根目录
3. 等待几分钟让部署生效
4. 清除浏览器缓存后重试

---

### Q12: 可以使用自定义域名吗？

**A:** 可以！所有平台都支持自定义域名。详细步骤请参考 DEPLOYMENT_GUIDE.md。

---

### Q13: 如何更新已部署的网站？

**A:** 取决于你的部署方式：

**Git 部署（GitHub Pages、Vercel、Netlify）**
- 修改代码
- 提交并推送到仓库
- 平台会自动重新部署

**拖拽部署（Netlify）**
- 修改本地文件
- 重新拖拽上传

## 功能问题

### Q14: 可以添加自己的图片吗？

**A:** 当然可以！
1. 将图片放入 `assets/images/` 文件夹
2. 在 `config.json` 或 `index.html` 中引用
3. 推荐使用 JPG 或 PNG 格式

---

### Q15: 支持多语言吗？

**A:** 支持！目前支持：
- 中文（zh）
- 英文（en）

在 `config.json` 中设置：
```json
{
  "config": {
    "language": "zh"
  }
}
```

---

### Q16: 可以添加博客吗？

**A:** EasyFolio 专注于作品集展示，不包含博客功能。但你可以：
- 在项目部分展示你的文章
- 链接到外部博客平台（Medium、掘金等）

---

### Q17: 如何添加联系表单？

**A:** 可以使用第三方服务：
- Netlify Forms（如果你用 Netlify 部署）
- Formspree
- Getform

这些服务都有免费额度。

## 技术问题

### Q18: 需要安装什么软件？

**A:** 基础使用只需要：
- 一个文本编辑器（VS Code、Sublime、记事本都可以）
- 一个浏览器

**可选**（用于高级功能）：
- Node.js（运行案例生成器）
- Git（版本控制）

---

### Q19: 案例生成器是干什么的？

**A:** 案例生成器是一个 Node.js 脚本，可以：
- 从 JSON 配置自动生成 HTML
- 提供多个模板选择
- 批量处理多个案例

这是可选的，你也可以直接编辑 HTML。

---

### Q20: 可以自定义样式吗？

**A:** 完全可以！
- 修改 `assets/css/style.css` 中的 CSS
- 添加自定义样式
- 修改配色方案
- 更换字体

---

### Q21: EasyFolio 收集我的数据吗？

**A:** 不收集！
- EasyFolio 是纯前端工具
- 所有数据都在你的本地或你自己的仓库
- 没有后端服务器
- 没有数据上报

---

### Q22: 我可以修改代码后商用吗？

**A:** 可以！EasyFolio 是开源的，请查看 LICENSE 文件了解具体许可条款。

---

### Q23: 遇到问题怎么寻求帮助？

**A:** 按以下顺序尝试：
1. 查看本文档（FAQ）
2. 查看其他文档（QUICKSTART、USER_MANUAL）
3. 检查是否有类似的 Issue
4. 提交新的 Issue
