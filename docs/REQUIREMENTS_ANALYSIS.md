## 1. 需求概述

### 1.1 项目背景

随着互联网求职和自由职业的发展，个人作品集已成为展示专业能力的重要方式。然而，传统作品集制作存在以下痛点：
- 技术门槛高，需要编程知识
- 制作周期长，需要大量时间投入
- 设计和排版成本高
- 缺乏专业的视觉效果

EasyFolio 旨在解决这些问题，提供一个零代码、快速上手的个人作品集生成工具。

### 1.2 商业目标

- **用户获取**: 吸引应届毕业生、自由设计师、内容创作者等目标用户
- **用户留存**: 通过优质体验和完整生态提升用户粘性
- **品牌建设**: 建立轻量级但专业的个人作品集工具品牌形象
- **生态建设**: 整合 Coze Agent、GitHub Pages、Vercel 等外部平台

### 1.3 关键成功因素

| 因素 | 重要性 | 说明 |
|------|--------|------|
| 文档质量 | ⭐⭐⭐⭐⭐ | 清晰详细的文档是降低使用门槛的关键 |
| 示例项目 | ⭐⭐⭐⭐⭐ | 高质量的示例项目让用户可以直接套用 |
| 部署便利性 | ⭐⭐⭐⭐ | 多平台支持，简单的部署流程 |
| AI辅助体验 | ⭐⭐⭐ | Coze Agent 集成提升内容生成效率 |
| 视觉设计 | ⭐⭐⭐⭐⭐ | 专业美观的界面提升展示效果 |

## 2. 用户需求分析

### 2.1 目标用户画像

- 年龄：22-25岁

- 技术能力：基本计算机操作，无编程经验

### 2.2 用户场景分析

#### 场景1：自由设计师接单

**用户**: 张明，UI设计师，30岁

**场景描述**:
张明接到一个新客户的咨询，客户要求看他的作品集。张明以前都是发PDF作品集，但效果不好。他需要快速创建一个专业的网页作品集。

**使用流程**:
1. 张明下载 EasyFolio
2. 按照快速入门指南，用30分钟完成配置
3. 添加3个代表性项目
4. 使用 Coze Agent 优化项目描述
5. 部署到 GitHub Pages
6. 分享链接给客户

**结果**:
客户通过网页作品集看到了张明的专业作品，对设计能力印象深刻，顺利签单。

## 3. 功能需求分析

### 3.1 核心功能模块

#### 模块1：AI简历分析

**功能描述**:
提供 Coze Agent 外部链接，用户可以跳转到 Coze Agent 进行简历分析，返回结构化数据模板。

**详细需求**:
1. 提供清晰的 Coze Agent 使用指南
2. 提供 Agent 配置提示词
3. 输出标准化的 JSON 数据格式
4. 指导用户如何保存和使用返回的数据

**验收标准**:
- [x] Coze Agent 使用指南文档完整
- [x] 数据格式模板提供
- [x] 用户能按指南完成简历分析

---

#### 模块2：项目作品管理

**功能描述**:
通过本地文件管理项目案例，支持添加、编辑、删除项目案例，支持项目描述、技术栈、成果展示、链接等信息展示。

**详细需求**:
1. 支持 JSON 和 Markdown 两种格式
2. 项目信息包含：
   - 项目名称
   - 项目描述
   - 技术栈
   - 项目日期
   - 项目链接
   - 项目图片
   - 思维导图
3. 提供项目数据格式模板
4. 提供多个示例项目

**验收标准**:
- [x] 数据格式模板提供
- [x] 3个示例项目提供
- [x] JSON/MD 格式支持

---

#### 模块3：一键生成静态网站

**功能描述**:
基于用户数据生成静态 HTML 网站，提供 Node.js 脚本支持。

**详细需求**:
1. Node.js 生成脚本
2. 支持批量生成多个案例
3. 生成速度 ≤ 10秒
4. 本地预览支持
5. 生成的 HTML 可直接部署

**验收标准**:
- [x] 生成脚本正常运行
- [x] 生成速度 ≤ 10秒
- [x] 生成的 HTML 可正常访问

---

#### 模块4：个人信息管理

**功能描述**:
通过本地配置文件编辑个人基本信息，包括姓名、职位、联系方式等。

**详细需求**:
1. 个人信息包含：
   - 姓名
   - 职位/专业身份
   - 邮箱
   - 电话（可选）
   - 所在城市（可选）
   - 个人简介
   - 社交媒体链接（LinkedIn、GitHub、作品集）
2. 教育经历管理
3. 工作经历管理
4. 技能管理（技能名称、级别、分类）

**验收标准**:
- [x] 数据格式模板提供
- [x] 字段说明完整

---

#### 模块5：首页展示

**功能描述**:
个人简介、技能展示、案例预览的首页展示功能。

**详细需求**:
1. 导航栏：
   - Logo
   - 导航链接（首页、案例、关于我）
   - 语言切换按钮
   - 移动端菜单按钮
2. Hero 区域：
   - 个人简介
   - 职业标签
   - 技能卡片
   - CTA 按钮
3. 案例展示区域：
   - 手风琴式案例卡片
   - 案例导航指示器
4. 方法论区域
5. 联系区域

**验收标准**:
- [x] 首页功能完整
- [x] 响应式设计正常
- [x] 交互效果流畅

---

#### 模块6：多语言支持

**功能描述**:
中英文切换、内容国际化支持。

**详细需求**:
1. 支持中文（zh）和英文（en）
2. 实时语言切换
3. HTML 标签支持多语言
4. 配置文件支持语言设置

**验收标准**:
- [x] 中英文切换正常

---

#### 模块7：交互体验

**功能描述**:
滚动动画、悬停效果、页面过渡等交互效果。

**详细需求**:

3. 手风琴交互：点击案例卡片展开详细信息
4. 页面过渡：页面跳转时的平滑过渡动画
5. 返回顶部：滚动时显示返回顶部按钮

**验收标准**:
- [x] 滚动动画流畅
