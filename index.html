Prompt工程 工作流设计

当一个品牌每周需要100条文案时

如何用AI重建内容生产工作流

品牌团队内容需求量持续增长，小红书/抖音/详情页三端同步更新，人工生产的效率和一致性都难以保证。核心矛盾：不是"AI能不能写文案"，而是"AI写出来的东西品牌敢不敢用"。

点击翻转揭示更多信息
背景

品牌团队内容需求量持续增长，小红书/抖音/详情页三端同步更新，人工生产的效率和一致性都难以保证。

核心矛盾：不是"AI能不能写文案"，而是"AI写出来的东西品牌敢不敢用"。调性失控的风险，比效率低更让品牌顾虑。

我的角色

负责整体方案设计、Prompt模板库搭建、工作流程优化及质量标准制定。

品牌对AI内容的信任，不是靠"证明AI很厉害"建立的，而是让第一批输出让团队"觉得可以用"。
问题拆解
内容生产问题 ──┬── 平台差异（小红书/抖音/微博格式不同）
               ├── 调性失控（多人写/AI写都可能失控）
               └── 效率瓶颈（选题→初稿→审核→发布，环节多）

AI介入的正确节点：
选题策划 → [初稿生成 ✓] → 人工审核 → [格式优化 ✓] → 发布
              ↑                              ↑
         AI最有价值的地方              AI可以标准化的地方
解决方案

① 梳理工作流，确定AI介入节点（不是全替代，是辅助关键环节）

② 大模型选型 → 营销文案场景基于调性适配度 + 输出稳定性综合选型

③ Prompt模板库设计：覆盖种草文案 · 产品详情 · 活动推广 · 节日借势

④ 质量验证标准：调性符合度 / 关键信息覆盖 / 格式规范 / 可读性

方法论沉淀

· 内容生成SOP文档（可复用于新场景）

· Prompt版本管理规范（记录每次迭代的修改逻辑）

· 新场景接入流程（从需求到上线的标准路径）

02
用户研究 需求分析

当用户说"不知道要什么"时

如何从模糊需求中提取明确洞察

产品初期，用户往往只能描述"痛点"而非"解决方案"。这时候需要一套方法，把"我觉得慢"转化为具体的性能指标，把"看起来乱"转化为信息架构的优化方向。

背景

面对10+个大模型，最常见的错误做法是"哪个最火用哪个"或"都试试感觉一下"。这两种方式都无法复现，也无法解释为什么做这个选择。

模型选型是产品决策，不是技术决策。选的不是"哪个更强"，而是"哪个在这个具体场景下最符合业务标准"。
四维评估框架
维度 权重 评估方法
品牌调性适配 35% 盲测输出与品牌风格匹配度（1-5分）
输出稳定性 30% 同一Prompt执行10次，计算方差
内容准确度 20% 事实类信息核查错误率
响应体验 15% 延迟、上下文长度、流畅度
多模型竞优评估法
测试设计：
① 准备3组真实营销任务（难度：低 / 中 / 高）
② 每组任务同时输入5个候选模型
③ 输出结果隐去模型来源，由2-3人独立盲评打分
④ 三轮淘汰，最终筛出2个最适合该场景的模型

核心逻辑：
不是比哪个模型"好"，而是在真实任务里淘汰不适合的。
场景化结论

小红书种草 → 豆包在情感化表达上更自然

产品详情页 → GPT-4在结构化信息组织上更稳定

创意短文案 → Deepseek在中文语感上有优势

产品策略 数据驱动

当功能越做越多却留不住用户时

如何做减法，重新定义核心价值

产品进入瓶颈期的典型表现：功能列表很长，但用户说"越来越难用"。这时候不是要做更多，而是要停下来，重新思考"用户真正雇我们做什么"。

背景

数据来源：移动电商平台开放数据集，1000万+条用户行为记录

核心问题：平台用户量大，运营资源有限，如何精准投入？

分析路径
原始数据（1000万条行为日志）
    ↓ MySQL 数据清洗 + 聚合
用户特征矩阵（每用户3个维度）
    ↓ RFM 建模
R（最近购买距今天数）· F（购买频率）· M（消费金额）
    ↓ K-means 聚类（K=5）
5个用户群体
    ↓ Python 可视化
分层报告 + 运营策略建议
分层结果
群体 占比 建议策略
高价值活跃12%VIP维护
高价值沉睡8%唤醒活动
潜力新用户20%培育引导
一般活跃35%常规运营
低价值沉睡25%低成本触达
关键发现

① 前20%用户贡献约68%消费额

② 唤醒成本 < 拉新成本的1/5

数据驱动 指标设计

当数据好看但用户骂声一片时

如何建立真正有用的指标体系

虚荣指标会欺骗我们：日活涨了可能是因为推送变多了，留存高了可能是因为卸载难了。我们需要一套"北极星+护栏"的指标组合，确保数据增长与用户体验一致。

点击翻转揭示更多信息
背景

数据团队每周都在发漂亮的报表，但产品团队觉得"数据和用户感受对不上"。

指标不是用来粉饰太平的，是用来发现问题的。如果指标好看但体验变差，那一定是指标设计错了。
虚荣指标 vs 真实指标
指标类型 问题 真实指标
日活（DAU）推送太多导致打开自然日活
7日留存卸载流程太复杂主动使用天数
转化率把入口藏得太深完成时长
北极星 + 护栏指标体系
北极星指标（定义成功）：
  产品核心价值 = 用户每周主动使用时长

护栏指标（防止走歪）：
  ① 推送打开率 < 30% → 说明打扰太多
  ② 卸载率 > 5% → 说明体验变差
  ③ 用户投诉量 ↑ → 说明哪里出问题了

决策逻辑：
  只有北极星↑ 且 护栏稳定 → 才是真的变好
